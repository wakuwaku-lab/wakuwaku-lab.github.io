Papertail

A high-performance CLI tool for downloading and processing logs from the Papertrail API.

By wakuwaku-lab / exia.huang

The Problem & Solution

  • Retrieving Papertrail logs can be slow and manual.
  • Papertail automates log fetching via the API.
  • Built with Go for maximum speed and portability.
  • Supports multiple output formats: TSV and SQLite.

Installation

go install github.com/wakuwaku-lab/papertail@latest

Or build from source:

git clone ... && go build -o papertail .

Configuration

  • Requires PAPERTRAIL_API_TOKEN.
  • Method 1: Environment variables.
  • Method 2: .env file in project root.
  • Seamlessly integrates into CI/CD and local environments.

Mode 1: Time Window

Fetch logs from a specific hour range:

# Last 24 hours
papertail logs --hours 24 --tsv out.tsv
# Hour range (1-6 hours ago)
papertail logs --hours 1-6 --tsv out.tsv

Mode 2 & 3: Dates

Specific Date:

papertail logs --date 2024-03-01 --tsv out.tsv

Date Range:

papertail logs --start 2024-03-01 --end 2024-03-07

Key Parameters

  • --hours (-n): Time window in hours.
  • --date (-D): Specific YYYY-MM-DD.
  • --tsv (-t): Path for TSV output.
  • --db (-d): SQLite database destination.

SQLite Analysis

  • Directly import logs into an SQLite database.
  • Enables standard SQL queries for complex analysis.
  • Command: papertail logs --db logs.db --tsv logs.tsv.
  • Perfect for persistent storage and reporting.
