# Agentic Project Template

Claude Codeのエージェンティックエンジニアリングを実証するテンプレートプロジェクトです。

## 概要

このプロジェクトは、記事「Claude Code エージェンティックエンジニアリング完全ガイド」で紹介されている3層アーキテクチャを実際に動作するコードとして実装したものです。

### アーキテクチャ

```
┌─────────────────────────────────────────────┐
│ CLAUDE.md (ディレクトリ)                    │
│   → セッションハーフック                     │
│   → ナビゲーション特化                        │
├─────────────────────────────────────────────┤
│ .claude/rules/ (条件付き)                   │
│   → 段階的開示 (Progressive Disclosure)    │
│   → pathsベースの注入                       │
├─────────────────────────────────────────────┤
│ skills/ (オーケストレーション)              │
│   → 遅延読み込み                             │
│   → Agents + Scripts + References          │
└─────────────────────────────────────────────┘
```

## インストール

```bash
npm install
```

## 実行

```bash
npm start
```

## デモ出力

```
=== Agentic Engineering Demo ===

[Layer] CLAUDE.md - Session Start Hook
  → Loaded as: User Message

[Conditional Rules] .claude/rules/
  → Progressive Disclosure: ON
  → paths: ["src/**", "apps/**"]

[Task Recipes] skills/
  → Lazy Loading: ON
  → Triggered on demand

✓ All layers initialized
```

## 主要概念

### 1. CLAUDE.mdはセッションハーフック

CLAUDE.mdはSystem Promptではなく、セッション開始時のUser Messageとして注入されます。ルールを書く場所ではなく、どのファイルを参照すべきかを記した「ディレクトリ」として機能させます。

### 2. Progressive Disclosure

条件付きルール（`.claude/rules/`）は、pathsで指定したファイルを初めてアクセスしたタイミングで注入されます。これにより、必要な情報が新しいメッセージとして届き、効果的になります。

### 3. スキルはオーケストレーター

スキルは「レシピ」ではなく、オーケストレーターとして機能します。SKILL.mdは「いつ・誰に・何を任せるか」だけを記述し、具体的な処理は外部ファイル（agents/, references/, scripts/）に委ねます。

### 4. 対立エージェントパターン

コード分析では、3つのエージェントを組み合わせて真実に近づきます：
- Bug Finder: ありとあらゆる問題を報告（過検出方向）
- Adversarial: 誤報を否定（過少検出方向）
- Referee: 中立的な最終判定

## プロジェクト構造

```
agentic-project-template/
├── CLAUDE.md                    # ディレクトリ
├── src/
│   ├── index.js                 # デモ実行
│   └── analyzer.js              # コード分析デモ
├── .claude/rules/
│   ├── meta/
│   │   └── execution-workflow.md
│   ├── coding/
│   │   └── coding-guides.md
│   └── writing/
│       └── writing-guides.md
└── skills/
    ├── writing-articles/
    │   └── SKILL.md
    └── analyzing-code/
        └── SKILL.md
```

在代码分析中，结合3个代理来接近真相：
- Bug Finder: 报告所有问题（过度检测方向）
- Adversarial: 否定误报（不足检测方向）

# Agentic Project Template

A template project demonstrating Claude Code Agentic Engineering.

## Overview

This project implements the 3-layer architecture introduced in the article "Claude Code Agentic Engineering Complete Guide" as runnable code.

### Architecture

```
┌─────────────────────────────────────────────┐
│ CLAUDE.md (Directory)                       │
│   → Session Start Hook                      │
│   → Navigation-focused                       │
├─────────────────────────────────────────────┤
│ .claude/rules/ (Conditional)               │
│   → Progressive Disclosure                  │
│   → paths-based injection                   │
├─────────────────────────────────────────────┤
│ skills/ (Orchestration)                    │
│   → Lazy Loading                            │
│   → Agents + Scripts + References           │
└─────────────────────────────────────────────┘
```

## Installation

```bash
npm install
```

## Run

```bash
npm start
```

## Demo Output

```
=== Agentic Engineering Demo ===

[Layer] CLAUDE.md - Session Start Hook
  → Loaded as: User Message

[Conditional Rules] .claude/rules/
  → Progressive Disclosure: ON
  → paths: ["src/**", "apps/**"]

[Task Recipes] skills/
  → Lazy Loading: ON
  → Triggered on demand

✓ All layers initialized
```

## Key Concepts

### 1. CLAUDE.md is Session Start Hook

CLAUDE.md is not a System Prompt but is injected as a User Message at session start. Instead of writing rules, it acts as a "directory" that specifies which files to reference.

### 2. Progressive Disclosure

Conditional rules (`.claude/rules/`) are injected when files matching `paths` are first accessed. This ensures essential information arrives as a newer message, making it more effective.

### 3. Skills are Orchestrators

Skills are not "recipes" but function as orchestrators. SKILL.md only describes "who to delegate what to and when", delegating specific processing to external files (agents/, references/, scripts/).

### 4. Adversarial Agent Pattern

In code analysis, combine 3 agents to approach truth:
- Bug Finder: Reports all possible issues (over-detection direction)
- Adversarial: Dismisses false positives (under-detection direction)
- Referee: Neutral final judgment
