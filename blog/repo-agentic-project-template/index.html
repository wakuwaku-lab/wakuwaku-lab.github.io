Agentic Project Template

Claude Codeのエージェンティックエンジニアリングを実証するテンプレートプロジェクトです。

概要

このプロジェクトは、記事「Claude Code エージェンティックエンジニアリング完全ガイド」で紹介されている3層アーキテクチャを実際に動作するコードとして実装したものです。

アーキテクチャ

┌─────────────────────────────────────────────┐
│ CLAUDE.md (ディレクトリ)                    │
│   → セッションハーフック                     │
│   → ナビゲーション特化                        │
├─────────────────────────────────────────────┤
│ .claude/rules/ (条件付き)                   │
│   → 段階的開示 (Progressive Disclosure)    │
│   → pathsベースの注入                       │
├─────────────────────────────────────────────┤
│ skills/ (オーケストレーション)              │
│   → 遅延読み込み                             │
│   → Agents + Scripts + References          │
└─────────────────────────────────────────────┘

インストール

npm install

実行

npm start

デモ出力

[Layer] CLAUDE.md - Session Start Hook
  → Loaded as: User Message

[Conditional Rules] .claude/rules/
  → Progressive Disclosure: ON
  → paths: ["src/**", "apps/**"]

[Task Recipes] skills/
  → Lazy Loading: ON
  → Triggered on demand

✓ All layers initialized

主要概念

1. CLAUDE.mdはセッションハーフック

CLAUDE.mdはSystem Promptではなく、セッション開始時のUser Messageとして注入されます。ルールを書く場所ではなく、どのファイルを参照すべきかを記した「ディレクトリ」として機能させます。

2. Progressive Disclosure

条件付きルール（.claude/rules/）は、pathsで指定したファイルを初めてアクセスしたタイミングで注入されます。これにより、必要な情報が新しいメッセージとして届き、効果的になります。

3. スキルはオーケストレーター

スキルは「レシピ」ではなく、オーケストレーターとして機能します。SKILL.mdは「いつ・誰に・何を任せるか」だけを記述し、具体的な処理は外部ファイル（agents/, references/, scripts/）に委ねます。

4. 対立エージェントパターン

コード分析では、3つのエージェントを組み合わせて真実に近づきます：

  • Bug Finder: ありとあらゆる問題を報告（過検出方向）
  • Adversarial: 誤報を否定（過少検出方向）
  • Referee: 中立的な最終判定
