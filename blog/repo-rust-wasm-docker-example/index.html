Rust WebAssembly Docker Example

This project demonstrates compiling Rust to WebAssembly and communicating with JavaScript, using Docker and docker-compose with mount volumes.

Prerequisites

  • Docker
  • docker-compose

Project Structure

rust-wasm-docker-example/
├── docker-compose.yml    # Docker compose with volume mounts
├── src/                 # Rust source code (mounted)
│   ├── Cargo.toml
│   └── src/lib.rs
└── www/                 # Web output (mounted)
    └── index.html

Quick Start

  1. Build the WASM module:
cd rust-wasm-docker-example
docker-compose run --rm wasm-builder
  1. Start the development server:
docker-compose up dev-server
  1. Open http://localhost:8080 in your browser

How It Works

  • wasm-builder service: Uses a pre-built Rust image with wasm-pack installed. Source code from ./src is mounted as a volume. After compilation, output goes to ./www/pkg.
  • dev-server service: Python HTTP server serves the www directory.
  • Volumes: Source code is mounted rather than copied into the image, following the principle of "prefer using mount volumes over building images when possible".

Features Demonstrated

  • Rust → JavaScript: Calling JS alert() from Rust
  • JavaScript → Rust: Calling Rust functions (greet, add, fibonacci) from JS
  • Bidirectional communication via wasm-bindgen

Building Manually

To build inside the container:

docker-compose run --rm wasm-builder

The compiled WASM will appear in www/pkg/.
