# Node.js PC Automation Example

This example demonstrates Node.js PC automation using the `robotjs` library, with docker-compose using mount volumes as required.

## Prerequisites

- Docker
- Docker Compose
- Node.js (for running outside container)

## Project Structure

```
node-pc-automation/
├── docker-compose.yml    # Docker Compose with volume mounts
├── Dockerfile           # Docker image definition
├── package.json         # Node.js dependencies
├── src/
│   └── index.js         # Automation script
├── README.md
└── THINKING.md
```

## Docker Compose with Volume Mounts

This project uses **mount volumes** instead of rebuilding images for code changes, following the best practice:

```yaml
volumes:
  - ./src:/app/src
  - ./package.json:/app/package.json
  - ./package-lock.json:/app/package-lock.json
```

This allows:
- Code changes in `src/` are reflected immediately
- No need to rebuild the Docker image for every code change
- Faster development iteration

## Running the Example

### Using Docker Compose

```bash
cd node-pc-automation
docker-compose up
```

### Running on Host System

For actual PC automation, run directly on the host:

```bash
npm install
npm start
```

**Note**: PC automation libraries like `robotjs` require direct hardware access and cannot function inside a Docker container. The script includes error handling to detect containerized environments.

## What It Does

1. Gets screen size
2. Moves mouse to center of screen
3. Performs mouse click
4. Types "Hello, World!"

## Dependencies

- `robotjs`: Node.js library for mouse and keyboard automation
